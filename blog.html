#Questions 01: Discuss The scope of var, let, and const

Answers:
var

Variables declared with var are function-scoped or globally-scoped, but not block-scoped.
If declared within a function, the variable is limited to that function's scope. If declared outside of any function, the variable is globally accessible throughout the entire script.
Variables declared with var are subject to "hoisting," meaning they are moved to the top of their scope during the compilation phase. Because of hoisting,
you can access and use a var variable before it's actually declared in the code.

let

Variables declared with let are block-scoped, meaning their scope is limited to the block (enclosed by curly braces) in which they are defined.
let variables are not hoisted to the top of their scope like var variables. They can be updated or reassigned within their scope.

const

Variables declared with const are block-scoped and also have a constant value, meaning they cannot be reassigned after their initial value assignment.
const variables must be assigned a value when declared, and this value cannot be changed. Like let, const variables are not hoisted to the top of their scope.

#Questions 02: Tell us the use cases of null and undefined

Answers:
null

It is the intentional absence of the value. It is one of the primitive values of JavaScript.

undefined

It means the value does not exist in the compiler. It is the global object.

#Questions 03: What do you mean by REST API?

Answers:
REST API

A REST API (Representational State Transfer Application Programming Interface) is a standardized way for software applications to communicate over the internet.
It uses HTTP methods (like GET, POST, PUT, and DELETE) to access and manipulate resources (such as data) on a server.
REST APIs are known for their simplicity, scalability, and use of URLs to identify resources.
They are widely used for building web services and enabling interactions between different software systems.
